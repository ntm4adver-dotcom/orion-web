# Orion Trading Bot — نسخة الويب (Render)

نسخة ويب من تطبيق **Orion Trading Bot** الأصلي (كان تطبيق أندرويد Kotlin)، تحافظ على نفس
استراتيجية التحليل الفعلية (**Explosive Breakout Hunter**) ونفس مصادر البيانات
(Binance Futures / OKX) ونفس تنبيهات تيليجرام، لكنها تعمل كخدمة ويب مستقلة 24/7
بدل الاعتماد على الجوال، مع لوحة تحكم على الويب لمتابعة الفحص والإشارات والتداول.

## ما الذي تم نقله بالضبط

- ✅ محرك التحليل الكمي الكامل (Bollinger, ATR, RSI, EMA, OBV Divergence، فلاتر الكسر
  الوهمي، وقف الخسارة الهيكلي...) — نفس المعادلات والعتبات الرقمية تماماً.
- ✅ جلب بيانات الشموع + Open Interest + Funding Rate + Order Book من Binance Futures و OKX.
- ✅ دورة الفحص التلقائي الدورية + متابعة حالة كل صفقة (PENDING → ACTIVE → HIT_TP/HIT_SL/CANCELLED).
- ✅ تنبيهات تيليجرام (إشارة جديدة + تحديثات الحالة) بنفس نص الرسائل تقريباً.
- ✅ لوحة ويب: شاشة رئيسية (تشغيل/إيقاف الفحص، سجل مباشر، جدول الإشارات)، إعدادات، وشاشة
  تداول لربط حساب OKX (رصيد، مراكز مفتوحة، تفعيل تنفيذ الصفقات تلقائياً).
- ✅ تسجيل دخول بكلمة مرور لحماية اللوحة.

## ما لم يُنقل في هذه النسخة الأولى (يمكن إضافته لاحقاً عند الطلب)

- محرك التطور الذاتي (Self-Evolution Engine) وتحليل Gemini AI للنصوص — غير مستخدم فعلياً
  في مسار التحليل الحالي أصلاً حسب كود التطبيق الأصلي (الاستراتيجية الوحيدة الفعالة هي
  Explosive Breakout Hunter).
- فلاتر القيمة السوقية (CoinGecko) وفلتر عملات الميم.
- المزامنة مع Firebase / نسخ Google Drive الاحتياطي.
- إشعارات الجوال (Push Notifications) — استُبدلت بلوحة ويب حية بدلاً منها.

## التشغيل محلياً

```bash
python -m venv .venv
source .venv/bin/activate   # على ويندوز: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env        # عدّل كلمة المرور والمفتاح السري بداخله
export $(cat .env | xargs)  # أو حمّل المتغيرات بالطريقة المناسبة لنظامك
uvicorn app.main:app --reload
```

افتح المتصفح على `http://localhost:8000` وسجّل الدخول بكلمة المرور التي وضعتها في `.env`.

## النشر على Render

1. من لوحة Render اختر **New > Blueprint** ثم اربط مستودع GitHub الذي رفعته.
   Render سيقرأ ملف `render.yaml` الموجود في المشروع تلقائياً وينشئ الخدمة.
2. عند الطلب، أدخل قيمة `ORION_APP_PASSWORD` (كلمة مرور لوحة التحكم). أما `ORION_SECRET_KEY`
   فسيتم توليده تلقائياً.
3. الخطة المستخدمة في `render.yaml` هي `starter` (خطة مدفوعة) وليست الخطة المجانية،
   لأن الخطة المجانية على Render **توقف الخدمة تلقائياً عند عدم النشاط** ولا تدعم قرص تخزين
   دائم — وكلاهما ضروري لتشغيل الفحص 24/7 وحفظ البيانات (الإشارات/الإعدادات) بشكل دائم بين
   عمليات إعادة التشغيل. إن كنت تريد البدء بالخطة المجانية للتجربة فقط، احذف قسم `disk`
   من `render.yaml` وغيّر `plan` إلى `free`، مع العلم أن البيانات ستُمسح وسيتوقف الفحص عند
   عدم وجود زيارات للوحة لفترة.
4. بعد اكتمال النشر، افتح رابط الخدمة الذي يعطيك إياه Render وسجّل الدخول بكلمة المرور.

### بديل: النشر يدوياً بدون Blueprint

من **New > Web Service**، اختر **Docker** كبيئة تشغيل، اربط المستودع، وأضف متغيرات البيئة
يدوياً (`ORION_APP_PASSWORD`, `ORION_SECRET_KEY`, `ORION_DB_PATH=/srv/data/orion.db`)
مع إضافة قرص (Disk) في تبويب **Disks** بمسار `/srv/data`.

## ⚠️ تنبيه مهم بخصوص التداول الآلي

قسم "التداول" يتيح ربط مفاتيح API الخاصة بحسابك في OKX وتفعيل تنفيذ صفقات **حقيقية**
تلقائياً دون تدخل يدوي عند رصد إشارة. هذا ينطوي على مخاطرة مالية حقيقية:

- ابدأ دائماً بتفعيل "وضع تجريبي (Demo/Testnet)" للتأكد من عمل كل شيء بشكل صحيح.
- مفاتيح API تُخزَّن في قاعدة البيانات المحلية للخدمة (غير مشفّرة حالياً) — احرص على أن
  كلمة مرور لوحة التحكم قوية، وأن الخدمة لا تُشارك مع أي شخص لا تثق به.
- يُفضّل إنشاء مفاتيح API بصلاحيات تداول فقط (بدون صلاحية سحب/تحويل الأموال) من إعدادات OKX.
